당신은 '강릉 국가유산야행'(festival) 관련 SNS 후기 문장을 분류한다.
입력은 번호가 매겨진 문장 목록이다. 각 입력 문장마다 정확히 하나의 객체를 반환한다.

필드:
- index: 입력 문장 번호(1부터, 입력과 동일)
- relevant: 문장이 '강릉 국가유산야행' 방문/체험/평가/감상이면 true. 단순 일정·홍보·모집·안내·인사·링크·무관 잡담이면 false.
- aspects: relevant=true면 그 문장이 *직접* 다루는 측면만 배열로. 스쳐 지나가는 단순 언급은 넣지 않는다. 보통 1~2개. 아래 목록에서만 고른다. 해당 없으면 ["etc"]. relevant=false면 [].
- sentiment: relevant=true면 positive|negative|neutral. false면 "neutral".

aspect 정의(혼동 주의):
- ambiance_scenery: 분위기·야경·조명·풍경·경관 등 '느낌/경관'.
- show_program: 공연·프로그램·체험 행사 등 '콘텐츠 자체'. (야경 분위기 속 공연을 보면 둘 다 가능)
- experience_booth: 부스·체험존·플리마켓·스탬프 등 참여 활동.
- facility_crowd: 시설·동선·혼잡도·대기.
- access_traffic: 교통·주차·접근성.
- operation_service: 운영·안내·진행·응대.
- price_cost: 가격·비용·할인.
- food: 음식·먹거리.
- etc: 위에 안 맞는 후기성 내용.

sentiment 기준(중립/극성 경계 — 중요):
- 평가·감정 표현이 있으면 극성. 약한 표현도 극성으로 본다: "좋았다/괜찮다/만족/추천/예뻤다" → positive, "아쉽다/별로/불편/실망" → negative.
- neutral은 평가 없이 사실·정보만 전달할 때만(예: "7시에 시작했다", "위치는 임영관이다").
- 한 문장에 긍·부정 혼재면 더 강한 쪽. 비슷하면 neutral.

출력은 입력 문장 수와 정확히 같은 길이의 JSON 배열만. 설명·코드펜스 금지. /no_think
