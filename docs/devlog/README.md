# Devlog 안내

## 목적

- `docs/devlog/`는 매일의 고민, 막힌 점, 실험, 선택지, 다음 액션을 남기는 작업 로그다.
- 아직 확정되지 않은 생각도 기록할 수 있다.
- 포트폴리오용 정제 문서보다, 실제 개발 과정과 판단 흔적을 남기는 데 초점을 둔다.

## 다른 문서와의 구분

| 위치 | 역할 |
| --- | --- |
| `docs/project_summary.md` | 현재 제품 정의와 실행 흐름을 짧게 보여주는 스냅샷 |
| `docs/devlog/` | 당일 고민, 챌린지, 실험, 다음 액션을 남기는 daily log |
| `docs/chat-notes/` | 확정된 결정 로그와 채팅 보관본 |

## 파일 규칙

- 파일명은 `YYYY-MM-DD.md`를 기본으로 쓴다.
- 하루에 여러 번 나눠 적더라도 가능하면 같은 날짜 파일에 누적한다.
- 최종 결정으로 굳어진 내용은 별도 결정 로그가 필요하면 `docs/chat-notes/`로 옮긴다.

## 권장 섹션

- 오늘의 질문
- 왜 이게 문제인가
- 시도한 것
- 관찰한 점
- 현재 판단
- 남은 리스크
- 다음 액션
- 참고 파일 / 실행 명령

## 작성 원칙

- 코드나 로그로 확인된 사실과 해석을 구분해서 쓴다.
- 확정되지 않은 내용은 `확인 필요:`로 적는다.
- 긴 설명보다 당일 판단에 필요한 근거와 다음 액션을 남긴다.
- 링크는 가능한 한 실제 파일, 테스트, 명령 기준으로 남긴다.
