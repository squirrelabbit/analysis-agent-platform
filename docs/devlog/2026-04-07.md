## 왜 이게 문제인가

- artifact presenter만으로는 사용자 질문 맥락에 맞는 자연어 답변을 만들기 어렵다.
- profile/prompt/rule 이름이 파일과 env에 흩어져 있으면 운영자가 오타나 drift를 늦게 발견하게 된다.
- `embedding_cluster`를 매 execution마다 다시 계산하면 dataset가 커질수록 IO, 메모리, 처리 시간이 모두 커진다.

## 시도한 것

- execution 완료 후 `result_v1`를 바탕으로 grounded `final_answer`를 생성하고 snapshot으로 저장하는 후처리 레이어를 추가했다.
- `smoke_final_answer.sh`와 worker 테스트를 통해 fallback 기반 회귀 검증을 붙였다.
- `dataset_profiles.json`, `config/prompts/*.md`, rule catalog를 함께 읽는 validator와 diagnostics 응답을 추가했다.
- `cluster`를 새 dataset build type으로 추가하고, `embedding_cluster`가 full-dataset 경로에서는 precomputed cluster artifact를 우선 읽도록 바꿨다.
